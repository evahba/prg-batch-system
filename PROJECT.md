# PRG Batch System — описание проекта

**Batch Caller 1.0** — внутренняя система управления приготовлением блюд для кухни Panda Express. Заменяет бумажные листы и устные команды. Шеф видит на экране, что готовить, в каком объёме и сколько времени осталось.

Живёт на `http://134.199.223.99`. Доступ без авторизации — внутренняя сеть.

---

## Экраны

Всего **5 экранов** + экран меню. Переключение скрытым свайпом (HiddenNav) в углу.

### Экран 1 — FOH (Front of House)

Экран для сотрудника на кассе. Здесь создаются заказы.

**Что видит:**
- Меню, сгруппированное по разделам (Chicken, Beef, Sides и т.д.)
- Кнопки выбора батча (1 / 2 / 3)
- Активные вызовы — тикеты, которые уже в работе
- Таймер обратного отсчёта по каждому активному тикету
- Индикатор Hold Time — сколько блюдо можно держать тёплым после готовности
- История завершённых тикетов за смену (по часам)

**Что делает:**
- Создаёт тикет — нажимает на позицию, выбирает батч → на кухонный экран уходит задача
- Отмечает приоритет (кнопка флага) — тикет мигает красным на всех экранах
- Отменяет тикет

**Комната WebSocket:** `foh`

## Таймеры

Каждый тикет в состоянии `started` имеет таймер обратного отсчёта.

### Как работает

1. Повар видит карточку в «Waiting» → нажимает **Start**
2. Бэкенд фиксирует `startedAt = now()`, запускает серверный `setTimeout`
3. На все экраны приходит событие `timer_started` с `startedAt` и `durationSeconds`
4. Клиент считает оставшееся время: `durationSeconds - (now - startedAt) / 1000`
5. Когда время вышло — сервер эмитит `timer_ended`, клиент показывает **QUALITY CHECK**

### Кнопки управления таймером

| Кнопка | Что делает |
|---|---|
| **Start** | Запускает таймер с нуля |
| **Reset** | Перезапускает таймер с начала (та же длительность) |
| **+10s** | Добавляет 10 секунд (доступна только на Quality Check) |
| **Step 2** | Добавляет 3 минуты (180 сек) — появляется вместо Complete при первом Quality Check |
| **Complete** | Завершает тикет, убирает с экрана |

### Двухшаговый таймер (Step 2)

Некоторые блюда требуют двух этапов:

1. Таймер заканчивается → появляется кнопка **Step 2** вместо Complete
2. Повар нажимает Step 2 → таймер добавляет +3 минуты, начинается второй этап
3. Второй таймер заканчивается → появляется **Complete**

Step 2 используется один раз на тикет.

### Синхронизация времени

Клиент и сервер синхронизируют время через `ping/pong`. Клиент вычисляет `offsetMs` и учитывает его при расчёте таймера. Это устраняет расхождения между часами устройств.

## Звуки

Реализованы через Web Audio API (без файлов, синусоида).

| Событие | Звук |
|---|---|
| Новый тикет пришёл на BOH-экран | Высокий пик 1200 Гц, 0.6 сек |
| Quality Check (таймер истёк) | Средний сигнал 880 Гц, 0.3 сек |

Звук при первом снапшоте не воспроизводится — только при новых заказах.

## Батчи

Каждая позиция поддерживает размеры батча: `1`, `2`, `3`.  
Рекомендуемый батч задаётся отдельно для каждого дейпарта (`recommendedBatch`).

Дейпарты:
- Breakfast: 6–11
- Lunch: 11–14
- Snack (Downtime): 14–17
- Dinner: 17–20
- Late Snack (Downtime): 20–24

---

## Hold Time

После завершения таймера блюдо можно держать тёплым ограниченное время. Значение хранится в `MenuItem.holdTime` (секунды, дефолт 600 = 10 мин).

На FOH-экране после завершения тикета отображается обратный отсчёт Hold Time — чтобы сотрудник знал, когда блюдо нужно выбросить.

---

## Приоритет

Любой тикет можно отметить как приоритетный. После этого:
- Карточка обводится красной рамкой
- Метка `Waiting` мигает красным
- Тикет поднимается вверх очереди в Waiting

Повторное нажатие снимает флаг.
